    **GREEN phase:** Add three functions to install.sh and wire them into the main flow:

    1. **merge_ralph_config():** Takes config_file path as argument.
       - Check if `.ralph` key exists: `jq -e '.ralph' "$config_file" >/dev/null 2>&1` -- if yes, print_info "Ralph config already present" and return 0
       - Validate JSON: `jq '.' "$config_file" >/dev/null 2>&1` -- if fails, print_error about invalid JSON and return 1
       - Create temp file in same directory: `mktemp "$(dirname "$config_file")/config.json.XXXXXX"`
       - Merge: `jq '. + {"ralph":{"enabled":true,"max_turns":50,"permission_tier":"default","timeout_minutes":30}}' "$config_file" > "$tmp_file"`
       - Atomic replace: `mv "$tmp_file" "$config_file"`
       - On jq failure: rm temp file, print_error, return 1
       - On success: print_success "Added ralph config to config.json"

    2. **verify_installation():** Takes no arguments (runs from target project root).
       - Loop over 4 script files checking -f and -x
       - Check 2 non-script files for -f only
       - Check `jq -e '.ralph' .planning/config.json`
       - Count and report errors
       - Return error count (0 = success)

    3. **print_summary():** Uses INSTALLED and SKIPPED counters.
       - Print a divider line
       - Print "Installation complete!" in bold green (or "Already up to date!" if INSTALLED=0)
       - Print "Installed: N files" if INSTALLED > 0
       - Print "Skipped: N files (already up to date)" if SKIPPED > 0
       - Print blank line
       - Print "Next steps:" section:
         - "1. Run /gsd:ralph execute-phase N --dry-run to verify"
         - "2. See .claude/skills/gsd-ralph-autopilot/SKILL.md for autopilot behavior rules"

    4. **Wire into main flow** (after the existing file copy calls):
       ```
       merge_ralph_config ".planning/config.json" || exit 1
       verify_installation || { print_error "Verification failed"; exit 1; }
       print_summary
       ```
