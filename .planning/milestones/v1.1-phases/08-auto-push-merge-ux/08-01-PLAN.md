Create `lib/push.sh` as a new Bash module (#!/bin/bash header, shellcheck directives).

The file must source `"$GSD_RALPH_HOME/lib/safety.sh"` is NOT needed here -- push.sh has no destructive operations. Do NOT source common.sh (output helpers are already loaded by the entry point, same pattern as safety.sh).

Implement two functions:

1. `has_remote()` -- Check if a pushable remote "origin" exists.
   - Use `git remote get-url origin >/dev/null 2>&1` (not `git remote show` which makes a network call).
   - Return 0 if origin exists, 1 if not.

2. `push_branch_to_remote()` -- Push a branch to origin, non-fatal.
   - Takes one argument: `branch_name`.
   - First, check `AUTO_PUSH` variable: if `"${AUTO_PUSH:-true}" == "false"`, print_verbose and return 0.
   - Second, check `has_remote`. If no remote, print_verbose and return 0.
   - Third, attempt `git push -u origin "$branch_name" >/dev/null 2>&1`.
   - On success: `print_success "Pushed $branch_name to origin"`.
   - On failure: `print_warning "Could not push $branch_name to origin (network issue or auth failure)"` and `print_warning "Branch is still available locally. Push manually with: git push origin $branch_name"`.
   - ALWAYS return 0 -- push failures must never crash the command.

Follow existing codebase conventions:
- `#!/bin/bash` shebang
- Comment header matching `lib/safety.sh` style: `# lib/push.sh -- Remote detection and push operations`
- No `set -euo pipefail` (not used in other lib/ modules)
- Use `print_info` for the "Pushing..." message before the attempt

**lib/config.sh** -- Append a new function `load_ralphrc()` AFTER the existing `detect_project_type()` function:

```bash
# Load project .ralphrc configuration into shell variables.
# Sources .ralphrc if it exists and passes bash -n syntax validation.
# Sets AUTO_PUSH default to "true" if not defined after loading.
# Returns: 0 always
load_ralphrc() {
    local ralphrc_path=".ralphrc"
    if [[ -f "$ralphrc_path" ]]; then
        if bash -n "$ralphrc_path" 2>/dev/null; then
            # shellcheck source=/dev/null
            source "$ralphrc_path"
        else
            print_warning ".ralphrc has syntax errors, using defaults"
        fi
    fi
    # Set defaults for any missing values
    AUTO_PUSH="${AUTO_PUSH:-true}"
    return 0
}
```

Key details:
- The `bash -n` check prevents crashes from malformed config files.
- `# shellcheck source=/dev/null` suppresses SC1091 for the dynamic source path (same pattern as other modules).
- `AUTO_PUSH` defaults to `"true"` -- push is opt-out, not opt-in.
