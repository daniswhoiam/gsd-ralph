<objective>
Make the merge command work from any branch state with any working tree state.

Purpose: Currently `gsd-ralph merge N` dies if not on main and dies if working tree is dirty. This plan replaces both hard failures with smart handling: auto-detect main branch and switch to it, auto-stash uncommitted changes before switching, and restore the stash after merge completes (success or rollback). Uses apply+drop pattern (not pop) so failed restoration preserves the stash entry.

Output: Updated `lib/commands/merge.sh` with auto-switch and auto-stash, updated `lib/merge/rollback.sh` with stash-aware rollback.
</objective>

**Step C: Replace the clean working tree check** (lines ~179-184) with auto-stash logic:

Replace this block:
```bash
# Verify clean working tree
local porcelain
porcelain=$(git status --porcelain 2>/dev/null)
if [[ -n "$porcelain" ]]; then
    die "Working tree is not clean. Commit or stash changes before merging."

```

With:
```bash
# Auto-stash dirty working tree
local porcelain
porcelain=$(git status --porcelain 2>/dev/null)
if [[ -n "$porcelain" ]]; then
    print_info "Uncommitted changes detected, auto-stashing..."
    if git stash push --include-untracked -m "gsd-ralph-merge-autostash" >/dev/null 2>&1; then
        did_stash=true
        print_success "Changes stashed"
    else
        die "Failed to stash uncommitted changes. Commit or discard them manually."
    fi
fi

# Switch to main if not already on it
if [[ "$current_branch" != "$main_branch" ]]; then
    print_info "Currently on '$current_branch', switching to $main_branch..."
    if ! git checkout "$main_branch" >/dev/null 2>&1; then
        # Restore stash before dying
        if [[ "$did_stash" == true ]]; then
            git stash apply >/dev/null 2>&1 && git stash drop >/dev/null 2>&1
        fi
        die "Failed to switch to $main_branch"
    fi
    print_success "Switched to $main_branch"
fi
```

Key details:
- Use `--include-untracked` with `git stash push` to capture untracked files (prevents "would be overwritten" errors during checkout).
- Stash BEFORE checkout (dirty tree blocks checkout).
- If checkout fails, restore stash immediately before dying.
- Use `git show-ref --verify` to detect main branch by existence, not by current branch name. This handles the case where user is on a completely unrelated branch.

**Step D: Add stash restoration at all exit points.** Find every `return` statement in `cmd_merge()` after the stash point and add stash restoration before it. The cleanest approach: add a helper function at the top of cmd_merge and call it before returns.

Add this local-scope helper right after the `did_stash` variable declaration:

Actually, since Bash doesn't have local functions, define `_restore_merge_stash()` at the top of the file (after the source lines but before merge_usage):

```bash
# Restore auto-stashed changes if stash was performed.
# Uses apply+drop (not pop) so failed apply preserves the stash entry.
# Args: none (uses global _MERGE_DID_STASH)
# Returns: 0 always
_restore_merge_stash() {
    if [[ "${_MERGE_DID_STASH:-false}" == true ]]; then
        print_info "Restoring auto-stashed changes..."
        if git stash apply >/dev/null 2>&1; then
            git stash drop >/dev/null 2>&1
            print_success "Stashed changes restored"
        else
            print_warning "Stash conflicts with merged changes. Your changes are safe in: git stash list"
            print_warning "Resolve with: git stash pop"
        fi
    fi
}
```

Modify `rollback_merge()` in `lib/merge/rollback.sh` to be aware of the auto-stash.

The `rollback_merge()` function does `git reset --hard "$saved_sha"`. After this reset, the stash can still be applied because stash entries are independent of the branch state.

Add stash restoration after the reset:

After the `print_success "Rolled back to pre-merge state: ${saved_sha}"` line, add:

```bash
# Restore auto-stash if cmd_merge stashed before merge
# _MERGE_DID_STASH is set by cmd_merge in merge.sh
if [[ "${_MERGE_DID_STASH:-false}" == true ]]; then
    print_info "Restoring auto-stashed changes..."
    if git stash apply >/dev/null 2>&1; then
        git stash drop >/dev/null 2>&1
        print_success "Stashed changes restored after rollback"
    else
        print_warning "Stash conflicts after rollback. Your changes are safe in: git stash list"
        print_warning "Resolve with: git stash pop"
    fi
fi
```

Also, note that `rollback_merge` is called early in cmd_merge (at the `--rollback` flag handling, line ~157-159), which is BEFORE auto-stash happens. In that code path, `_MERGE_DID_STASH` will be false/unset, so the stash restore block will be a no-op. This is correct behavior: a standalone `gsd-ralph merge N --rollback` should not attempt stash restoration.

However, rollback can ALSO be invoked by the user AFTER a failed merge (they run `gsd-ralph merge N --rollback` as a separate command). In that case, the stash from the PREVIOUS merge invocation is gone from `_MERGE_DID_STASH` (it was a different process). The user would need to manually `git stash pop` in that case, which is correct -- we cannot track stash state across processes. The stash message "gsd-ralph-merge-autostash" makes it identifiable.

No other changes to rollback.sh are needed.
