1. `@test "execute pushes branch to remote when origin exists"`:
   - Set up a bare repo as remote: `git init --bare "$BATS_TEST_TMPDIR/remote.git"`
   - Add it as origin: `git remote add origin "$BATS_TEST_TMPDIR/remote.git"`
   - Create a minimal phase with a plan file
   - Run `gsd-ralph execute N`
   - Verify the branch exists on the remote: `git -C "$BATS_TEST_TMPDIR/remote.git" branch -l` should list the phase branch
   - OR verify push happened: check output contains "Pushed"

2. `@test "execute skips push when no remote"`:
   - Do NOT add a remote
   - Create a minimal phase with a plan file
   - Run `gsd-ralph execute N`
   - Verify the command succeeds (exit 0)
   - Verify output does NOT contain "Pushed"

3. `@test "execute skips push when AUTO_PUSH=false"`:
   - Add a remote
   - Set `AUTO_PUSH=false` in .ralphrc
   - Create a minimal phase with a plan file
   - Run `gsd-ralph execute N`
   - Verify the command succeeds
   - Verify the branch does NOT exist on the remote

**tests/merge.bats -- Add these tests at the end of the file:**

4. `@test "merge auto-switches to main from phase branch"`:
   - Set up a test repo on main with a phase branch that has commits
   - Switch to the phase branch: `git checkout phase-N/slug`
   - Run `gsd-ralph merge N`
   - Verify merge succeeds
   - Verify current branch is now main: `git symbolic-ref --short HEAD` == "main"
   - Verify output contains "Switched to main" or similar

5. `@test "merge auto-stashes dirty working tree"`:
   - Set up a test repo on main with a phase branch
   - Create a dirty file: `echo "dirty" > untracked.txt`
   - Run `gsd-ralph merge N`
   - Verify merge succeeds
   - Verify the dirty file still exists: `[[ -f untracked.txt ]]`
   - Verify output contains "auto-stashing" or "stashed"

6. `@test "merge auto-stashes and restores from non-main branch"`:
   - Set up a test repo with a phase branch
   - Switch to phase branch
   - Create dirty changes
   - Run `gsd-ralph merge N`
   - Verify merge succeeds
   - Verify dirty changes are restored
   - Verify on main branch

7. `@test "merge pushes main to remote after successful merge"`:
   - Set up a bare remote repo, add as origin
   - Push initial main to remote
   - Create and execute a phase (create branch, make commits)
   - Switch back to main
   - Run `gsd-ralph merge N`
   - Verify the remote main has the merged commits: compare SHAs

8. `@test "merge skips push when AUTO_PUSH=false"`:
   - Set up remote, push initial main
   - Record remote main SHA before merge
   - Set `AUTO_PUSH=false` in .ralphrc
   - Run `gsd-ralph merge N`
   - Verify remote main SHA is unchanged (push was skipped)
