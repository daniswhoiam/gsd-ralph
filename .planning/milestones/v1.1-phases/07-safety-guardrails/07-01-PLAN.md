  <action>
Create a new file `lib/safety.sh` with two functions:

**safe_remove(target_path, [removal_type]):**
- Takes a target path and optional removal_type ("file" or "directory", default "file")
- Guard checks in order:
  1. Block empty/unset path (`[[ -z "$target_path" ]]`) -- print_error "SAFETY: Refusing to remove empty path", return 1
  2. Resolve to absolute path: if directory exists, use `cd "$target_path" && pwd -P`; for files, resolve parent dir with `cd "$(dirname "$target_path")" && pwd -P` then append basename
  3. Block filesystem root (`[[ "$abs_path" == "/" ]]`) -- print_error, return 1
  4. Block home directory (`[[ -n "${HOME:-}" ]] && [[ -d "$abs_path" ]] && [[ "$abs_path" -ef "$HOME" ]]`) -- print_error, return 1
  5. Block git toplevel (`git rev-parse --show-toplevel` then `[[ "$abs_path" -ef "$git_toplevel" ]]`) -- print_error, return 1
- If all checks pass: `rm -rf "$target_path"` for directory, `rm -f "$target_path"` for file
- Return 0 on successful removal, 1 on refusal

Use `[[ -ef ]]` for all path identity checks (inode-level, handles symlinks). Use `${HOME:-}` to avoid unset variable errors under `set -u`.

**validate_registry_path(path):**
- Block empty path -- print_error, return 1
- Allow sentinel `__MAIN_WORKTREE__` -- return 0 (valid but non-removable)
- Block non-absolute paths (must start with `/`) -- print_error, return 1
- Block path traversal (contains `..`) -- print_error, return 1
- Return 0 if valid

File header should match existing convention: `#!/bin/bash` comment block with filename and purpose. Do NOT source common.sh from safety.sh -- the sourcing chain is handled by the entry point (bin/gsd-ralph sources common.sh, then commands source safety.sh).

The file must be Bash 3.2 compatible (no associative arrays, no nameref, no `readlink -f`).
  </action>

Run `bash -n lib/safety.sh` -- expect zero syntax errors (Bash parse check).
Verify file has both function definitions: `grep -c '^safe_remove\|^validate_registry_path' lib/safety.sh` should output 2.
  </verify>
