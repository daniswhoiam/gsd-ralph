Create `benchmarks/harness/lib/metrics.sh` with Bash 3.2 compatibility (`#!/bin/bash`, `set -euo pipefail`).

**Source guard:** Same pattern as common.sh -- prevent direct execution.

**Function `extract_metrics()`:**
- Argument: `json_file` (path to claude -p JSON output file)
- Validate JSON is parseable: `if ! jq empty "$json_file" 2>/dev/null; then echo '{"error": "invalid_json"}'; return 1; fi`
- Extract with defensive fallbacks using jq:
  ```
  jq '{
      session_id: (.session_id // "unknown"),
      total_cost_usd: (.total_cost_usd // 0),
      duration_ms: (.duration_ms // 0),
      duration_api_ms: (.duration_api_ms // 0),
      num_turns: (.num_turns // 0),
      is_error: (.is_error // false),
      result_text: (.result // "")
  }' "$json_file"
  ```
- Outputs the extracted JSON object on stdout

**Function `parse_eval_score()`:**
- Argument: `eval_output` (string -- full stdout from bench-eval.sh)
- Extract the `Score: X/Y checks passed` line via: `score_line=$(echo "$eval_output" | grep -E '^Score: [0-9]+/[0-9]+') || score_line=""`
- If empty, echo "0" and return
- Parse passed: `passed=$(echo "$score_line" | sed -E 's/^Score: ([0-9]+)\/[0-9]+.*/\1/')`
- Parse total: `total=$(echo "$score_line" | sed -E 's/^Score: [0-9]+\/([0-9]+).*/\1/')`
- If total is 0, echo "0" and return
- Compute integer percentage: `echo $(( (passed * 100) / total ))`

**Function `capture_shellcheck_baseline()`:**
- Argument: `workdir` (path to benchmarks/taskctl/ in worktree)
- If shellcheck not available, echo "null" and return
- Count warnings: `count=$(shellcheck -f json "$workdir/src/"*.sh "$workdir/src/commands/"*.sh 2>/dev/null | jq 'length' 2>/dev/null) || count=0`
- Echo `"${count:-0}"`

**Function `capture_shellcheck_post()`:**
- Same as baseline. Separate function name for clarity. Identical implementation.

**Function `count_commits()`:**
- Arguments: `worktree_path` (full worktree root, not taskctl dir), `starting_tag`
- `git -C "$worktree_path" log --oneline "${starting_tag}..HEAD" 2>/dev/null | wc -l | tr -d ' '`

**Function `check_conventional_commits()`:**
- Arguments: `worktree_path`, `starting_tag`
- Count total commits. If 0, echo "true" and return.
- Count non-conventional: `bad=$(git -C "$worktree_path" log --format='%s' "${starting_tag}..HEAD" 2>/dev/null | grep -cvE '^(feat|fix|chore|docs|test|refactor|style|perf|build|ci)(\(.+\))?: .+') || bad=0`
- Echo "true" if bad == 0, else "false"

**Function `count_tests_added()`:**
- Arguments: `worktree_path`, `starting_tag`
- Count new .bats files: `git -C "$worktree_path" diff --name-only --diff-filter=A "${starting_tag}..HEAD" 2>/dev/null | grep -c '\.bats$'` with `|| echo 0` fallback
- This is a rough proxy; exact test function count is harder to capture reliably

Per research Pitfall 1: Do NOT attempt to extract input_tokens/output_tokens from --output-format json. Use num_turns and total_cost_usd instead. Set tokens_input/tokens_output to 0 in result schema.

  <name>Task 2: Create CC mode invocation script (cc.sh)</name>
  <files>benchmarks/harness/lib/modes/cc.sh</files>
  <action>
Create `benchmarks/harness/lib/modes/cc.sh` with Bash 3.2 compatibility (`#!/bin/bash`, `set -euo pipefail`).

**Source guard:** Same pattern -- prevent direct execution.

**Comment block at top documenting the mode contract:**
```
# Mode Contract:
# All mode scripts must implement mode_invoke() with this signature:
#   mode_invoke(prompt, workdir, max_turns, time_cap_seconds)
#     - stdout: JSON from the mode's execution
#     - stderr: redirected to workdir/.bench-stderr.log
#     - exit code: 0 = normal, 124 = timeout, other = error
#     - side effects: modifies files in workdir
```

**Function `mode_invoke()`:**
- Arguments: `prompt` ($1), `workdir` ($2), `max_turns` ($3), `time_cap_seconds` ($4)
- Validate all 4 arguments are non-empty, exit 1 with usage message if missing
- `cd` into `$workdir` before invoking claude (per research Pitfall 5: limits Claude's default scope to benchmarks/taskctl/)
- Redirect stderr to `"$workdir/.bench-stderr.log"`
- Invoke: `timeout "$time_cap_seconds" claude -p "$prompt" --output-format json --max-turns "$max_turns" --permission-mode auto --no-session-persistence 2>"$workdir/.bench-stderr.log"`
- Let the exit code propagate naturally (timeout returns 124, claude returns its own exit code otherwise)
- Do NOT catch the exit code inside mode_invoke -- let the caller (bench-run.sh) handle it
- The function should be a thin wrapper. No metric extraction, no score parsing. Just invoke and return.

**Key design decisions (from research):**
- Use `--permission-mode auto` not `--dangerously-skip-permissions` -- safer and sufficient
- Use `--no-session-persistence` to avoid accumulating session files during benchmark runs
- The `cd` into workdir is critical: it scopes Claude to the taskctl directory, preventing modifications outside benchmarks/taskctl/
- GNU timeout at `/opt/homebrew/bin/timeout` -- use `timeout` directly since it should be on PATH via Homebrew. If not found at runtime, the `require_command timeout` call in bench-run.sh (Plan 03) will catch it early.

Create the directory structure: `mkdir -p benchmarks/harness/lib/modes/`

Make the file NOT executable (it's sourced, not run directly).
