<task type="auto">
  <name>Task 2: Create worktree isolation script (bench-reset.sh)</name>
  <files>benchmarks/harness/bench-reset.sh</files>
  <action>
Create `benchmarks/harness/bench-reset.sh` with Bash 3.2 compatibility (`#!/bin/bash`, `set -euo pipefail`).

**Source common.sh:** `source "$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)/lib/common.sh"`

**Function `create_run_worktree()`:**
- Arguments: `run_id` (string), `starting_tag` (string)
- Compute `worktree_path="${BENCH_TMPDIR}/bench-run-${run_id}"`
- Run `git -C "$BENCH_REPO_ROOT" -c advice.detachedHead=false worktree add --detach "$worktree_path" "$starting_tag" 2>/dev/null`
- If worktree creation fails, log error and return 1
- Compute `workdir="$worktree_path/benchmarks/taskctl"`
- Validate starting state: check `[[ -f "$workdir/src/taskctl.sh" ]]`, if missing log error and return 1
- Run `git -C "$worktree_path" clean -fdx 2>/dev/null` for safety
- Initialize submodules: `git -C "$worktree_path" submodule update --init --recursive 2>/dev/null || true` (needed for Bats helpers per Phase 21 lessons)
- Echo `"$workdir"` on stdout (this is the return value)
- Log info: "Created worktree at $worktree_path (tag: $starting_tag)"

**Function `cleanup_run_worktree()`:**
- Arguments: `run_id` (string)
- Compute `worktree_path="${BENCH_TMPDIR}/bench-run-${run_id}"`
- Run `git -C "$BENCH_REPO_ROOT" worktree remove --force "$worktree_path" 2>/dev/null || true`
- Log info: "Cleaned up worktree for run $run_id"

**CLI mode (when run directly, not sourced):**
If script is executed directly (not sourced), parse arguments:
- `bench-reset.sh create <run_id> <starting_tag>` calls `create_run_worktree` and prints workdir
- `bench-reset.sh cleanup <run_id>` calls `cleanup_run_worktree`
- `bench-reset.sh --help` prints usage
- Any other arguments: print usage and exit 1

Use the pattern: `if [[ "${BASH_SOURCE[0]}" == "$0" ]]; then ... fi` for CLI mode.

Make the file executable: `chmod +x benchmarks/harness/bench-reset.sh`

Per research Pitfall 3: worktrees are in /tmp, outside the repo, avoiding .gitignore interference.
Per research Pitfall 4: submodule init ensures Bats helpers resolve in worktree context.
