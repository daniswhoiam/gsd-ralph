**Unit tests for safe_remove() (SAFE-02):**

1. `safe_remove rejects empty path` -- call `safe_remove ""` -- assert failure (return 1), assert output contains "SAFETY"
2. `safe_remove rejects filesystem root` -- call `safe_remove "/" "directory"` -- assert failure, assert output contains "filesystem root"
3. `safe_remove rejects home directory` -- Create a temp dir, set HOME to it, call `safe_remove "$HOME" "directory"` -- assert failure, assert output contains "home directory". Restore HOME after.
4. `safe_remove rejects git toplevel` -- In the test repo created by setup, call `safe_remove "$(git rev-parse --show-toplevel)" "directory"` -- assert failure, assert output contains "project root"
5. `safe_remove allows regular file deletion` -- create a temp file in test dir, call `safe_remove "$tempfile" "file"` -- assert success, assert file no longer exists
6. `safe_remove allows subdirectory deletion` -- create a subdirectory in test repo, call `safe_remove "$subdir" "directory"` -- assert success, assert directory no longer exists
7. `safe_remove rejects git toplevel via symlink` -- create a symlink to the test repo root, call `safe_remove "$symlink" "directory"` -- assert failure (symlink resolves to toplevel, caught by -ef)

**Unit tests for validate_registry_path() (SAFE-02, SAFE-03):**

8. `validate_registry_path rejects empty path` -- assert failure
9. `validate_registry_path accepts __MAIN_WORKTREE__ sentinel` -- assert success
10. `validate_registry_path rejects relative path` -- call with "relative/path" -- assert failure
11. `validate_registry_path rejects path traversal` -- call with "/some/../../../etc" -- assert failure
12. `validate_registry_path accepts absolute path` -- call with "/tmp/some-worktree" -- assert success

Use `run` for all function calls to capture output and exit status. Use `assert_failure` / `assert_success` / `assert_output --partial` from bats-assert.

Note: For the HOME directory test, save and restore the original HOME value. The -ef check requires the directory to actually exist, so use the mktemp-created directory.
