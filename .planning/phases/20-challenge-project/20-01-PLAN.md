Create the directory structure `benchmarks/taskctl/src/commands/`.

**storage.sh** -- JSON file storage layer using jq:
- `STORAGE_FILE` defaults to `${TASKCTL_DATA:-.taskctl.json}` (env var override for test isolation)
- `storage_read_all()` -- reads JSON array from file, returns `[]` if file missing
- `storage_add()` -- accepts description string, generates next ID via `storage_next_id`, creates timestamp with `date -u +%Y-%m-%dT%H:%M:%SZ`, appends to JSON array, writes file
- `storage_next_id()` -- reads max ID from existing tasks, returns max+1 (uses `jq '[.[].id] | max // 0'`)
- Do NOT include `set -euo pipefail` in storage.sh (it is sourced, not executed directly)

**taskctl.sh** -- CLI entry point:
- Shebang: `#!/bin/bash`
- `set -euo pipefail`
- Resolves `SCRIPT_DIR` from `BASH_SOURCE[0]` using `cd "$(dirname "${BASH_SOURCE[0]}")" && pwd`
- Sources `storage.sh` and `format.sh` from `$SCRIPT_DIR/`
- Case statement dispatching on first arg: add -> `cmd_add`, list -> `cmd_list`, done -> `cmd_done`
- Each case shifts then sources the command file from `$SCRIPT_DIR/commands/` and calls the function with remaining args
- Default case prints usage and exits 1
- Make executable (`chmod +x`)

**commands/add.sh** -- Add command:
- `cmd_add()` function that validates at least one arg exists (print error + exit 1 if missing)
- Joins all args as the description: `local description="$*"`
- Calls `storage_add "$description"`
- Prints confirmation message: "Added task N: description"

**commands/list.sh** -- List command:
- `cmd_list()` function
- Reads tasks via `storage_read_all`
- If `$1` is `--done`, call `format_task_list "$tasks" "done"`
- If `$1` is `--pending`, call `format_task_list "$tasks" "pending"`
- Otherwise, call `format_task_list "$tasks" "all"`
- Handles empty task list: if jq length is 0, print "No tasks found" and return

**commands/done.sh** -- Mark task done WITH PLANTED BUG (CHAL-02):
- `cmd_done()` function that accepts a task_id argument
- Validates task_id is provided (print error + exit 1 if missing)
- Reads all tasks via `storage_read_all`
- THE BUG: Uses `jq --argjson idx "$task_id" '.[$idx].done = true'` which treats the task ID as a 0-based array index instead of looking up by the `.id` field. So `taskctl done 3` marks `tasks[3]` (the 4th element, task with id=4) instead of the task with id=3.
- Writes updated JSON to `$STORAGE_FILE`
- Prints "Task $task_id marked as done" (message looks correct even though wrong task is modified)
- Do NOT add comments like "BUG" or "WRONG" -- the code should look natural, like a genuine developer mistake

**format.sh** -- Output formatting WITH DELIBERATE CODE SMELLS (CHAL-04):
- Must be functional (produces correct output) but messy
- Write it CLEAN first mentally, then apply these specific smells:

Smell catalog (all 9 must be present):
1. **Long function:** `format_task_list()` should be 35+ lines, doing filtering AND formatting in one function
2. **Poor variable names:** Use `t` for description, `d` for done status, `s` for created timestamp, `cnt` for counter
3. **Unquoted variables:** `echo $tasks | jq length` (not `echo "$tasks"`) -- produces SC2086
4. **Unquoted loop variables:** `while [ $i -lt $total ]` instead of `while [ "$i" -lt "$total" ]`
5. **Declare-and-assign in one line:** `local id=$(echo $tasks | jq -r ".[$i].id")` -- produces SC2155
6. **Duplicated formatting blocks:** The if/elif/else for done/pending/all should have copy-pasted printf statements with minor variations
7. **Misleading counter:** `done_cnt` variable that increments for ALL tasks, not just done ones
8. **Duplicated summary logic:** Separate echo blocks for done/pending/all counts at the bottom
9. **Duplicated function:** `format_single_task()` that repeats the same iteration and printf logic from `format_task_list()`

Use the code pattern from the research (Section "Code Smells for format.sh") as the reference implementation. The format should use `printf "[%s] #%s %s (%s)\n"` with "x" for done and " " for pending.

CRITICAL: format.sh must still produce CORRECT output. The smells are about style, not correctness. Verify mentally that the output matches what test_list.bats will expect.
